# web-code
我构建了一个基于多 Agent 协作的小米智能家居意图理解与自动化系统。核心痛点：现有智能家居控制依赖用户逐设备精确操作，缺乏对模糊意图的深层理解能力，导致"我要睡了"这类自然指令无法被正确拆解执行。

系统架构包含三层 Agent 协作：意图解析 Agent 负责将模糊自然语言指令通过长链推理拆解为结构化子任务链（如"睡觉"→灯光关闭、空调调至26度睡眠模式、窗帘关闭、安防启动）；设备调度 Agent 根据子任务匹配米家生态设备并生成执行序列，处理设备间依赖关系与冲突消解（如空调和加湿器的联动策略）；反馈验证 Agent 监控执行结果，若设备离线或执行失败则触发降级策略并通知用户。

该系统已基于米家开放平台完成原型验证，支持跨品类20+设备协同，场景意图识别准确率达92%，将多设备联动配置从平均2分钟手动操作压缩至3秒内一键响应，显著降低了智能家居的使用门槛。整体采用长链推理+多Agent协作的技术路线，实现了从"被动控制"到"主动理解"的体验跃升。




项目结构：
- index.html - 主界面
- css/style.css - 样式
- js/agents.js - Agent核心逻辑
- js/intent-parser.js - 意图解析Agent
- js/device-scheduler.js - 设备调度Agent
- js/feedback-verifier.js - 反馈验证Agent
- js/main.js - 主控制器



🏗️ 核心架构

用户输入 "我要睡了"
        │
        ▼
┌─────────────────────────────────────┐
│  Phase 1: 意图解析 Agent (7步推理)    │
│  • 关键词匹配 → 场景消歧 → 子任务拆解 │
│  • 输出：10个结构化原子操作            │
│  • 示例："关闭吸顶灯" / "空调26°C"     │
└─────────────┬───────────────────────┘
              │ tasks[]
              ▼
┌─────────────────────────────────────┐
│  Phase 2: 设备调度 Agent (3种冲突规则)│
│  • 同设备并发冲突 → 合并操作          │
│  • 电源开关互斥 → 取最终状态          │
│  • 依赖关系排序 → 拓扑重排            │
│  • 输出：有序执行序列                  │
└─────────────┬───────────────────────┘
              │ executionResults[]
              ▼
┌─────────────────────────────────────┐
│  Phase 3: 反馈验证 Agent (4种降级策略)│
│  • 逐项校验执行结果                   │
│  • 失败项触发降级（替代/重试/通知）     │
│  • 综合评分 → 生成执行报告             │
└─────────────────────────────────────┘
